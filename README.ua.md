# Домашнє завдання

- Створіть репозиторій з назвою **goit-react-hw-03**.
- При здачі домашньої роботи надайте два посилання:
  - на репозиторій з вихідними файлами
  - на робочу сторінку завдання на **Vercel**
- Проєкт повинен бути створений за допомогою [**Vite**](https://vite.dev/)
- Для створення форм використовуйте бібліотеку [**Formik**](https://formik.org/)
- Під час запуску коду завдання в консолі не повинно бути помилок або попереджень
- Для кожного компонента у папці `src/components` є окрема папка, яка містить JSX файл самого React компонента та файл його стилів
- Назви папки, файла компонента (з розширенням `.jsx`) та файла стилів (перед `.module.css`) однакові та відповідають назвам, вказаним у завданнях
- Для експорту компонентів використовується **export default**
- JS-код має бути чистим і зрозумілим, використовуйте **Prettier**
- Стилізація повинна бути виконана за допомогою **CSS-модулів**

## Компоненти

В цьому завданні інтерфейс вже розділений на компоненти.  
Твоя задача — продумати стан і пропси та перенести це в код.

![preview](./src/assets/goit-hw-react-3-1.png)

Частини інтерфейсу, які входять в компонент, відзначені рамкою відповідного кольору.

Потрібно виділити такі компоненти:

- кореневий компонент `App`
- форма додавання контактів `ContactForm`
- список контактів `ContactList`
- елемент списку контактів `ContactListItem`
- фільтр пошуку `SearchBox`

Як бачиш, всі компоненти рендеряться всередині компонента `App`.  
Його розмітка виглядатиме так (додатково передбачені пропси та інший код):

```jsx
<div>
  <h1>Phonebook</h1>
  <ContactForm />
  <SearchBox />
  <ContactList />
</div>
```

На цей раз ми будемо давати менше підказок, ніж у попередньому домашньому завданні.  
Це зроблено спеціально, щоб ти поступово звик до розробки компонентів, їх пропсів і логіки додатка.

## Крок 1 — Список контактів

У стані компонента `App` повинен бути масив контактів.  
На етапі розробки буде зручно жорстко закодувати деякі дані в стані.  
Це уникне необхідності ручного введення даних у інтерфейсі для тестування нового функціоналу.

Наприклад, використай цей масив контактів для початкового значення стану:

```js
[
  { id: "id-1", name: "Rosie Simpson", number: "459-12-56" },
  { id: "id-2", name: "Hermione Kline", number: "443-89-12" },
  { id: "id-3", name: "Eden Clements", number: "645-17-79" },
  { id: "id-4", name: "Annie Copeland", number: "227-91-26" },
];
```

Створіть компоненти **ContactList** та **Contact** і використовуйте їх для відображення списку контактів.  
Передайте необхідні значення через пропси.

![preview](./src/assets/goit-hw-react-3-2.png)

Після цього кроку інтерфейс застосунку буде виглядати наступним чином — заголовок сторінки та список контактів.

## Крок 2 — Пошук за ім'ям

Додайте поле пошуку **SearchBox**, яке можна використовувати для фільтрації списку контактів за ім'ям.

- Поле пошуку — це інпут без форми, значення якого записується у стан (контрольований елемент).
- Логіка фільтрації повинна бути нечутливою до регістру.

Стан фільтру слід зберігати в компоненті `App` і передавати потрібні значення як пропси компоненту **SearchBox**.  
Тоді фільтрація масиву контактів буде виконуватися в компоненті `App`, а її результат — масив відфільтрованих контактів — передавати пропсом компоненту **ContactList**.

Після цього кроку при вводі тексту в поле пошуку список контактів оновлюватиметься, щоб відобразити лише підходящі контакти.

## Крок 3 — Додавання контактів

Реалізуй додавання контакту до списку контактів через форму **ContactForm**.  
Кожен контакт повинен бути об'єктом з властивостями `name`, `number` та `id`, як у тестових даних, що зараз знаходяться у стані.

- Для генерації ідентифікаторів використовуй будь-який відповідний пакет, наприклад `nanoid`.
- На даному етапі база даних відсутня, тому ідентифікатор об'єкта створюється самостійно, наприклад, під час сабміту форми.

Форму обов'язково створи за допомогою бібліотеки **Formik**.  
Додай валідацію полів форми бібліотекою **Yup** та виводь повідомлення про помилки:

- Поля обов'язкові для заповнення
- Мінімальна кількість символів — 3
- Максимальна кількість символів — 50

Після завершення цього кроку застосунок повинен виглядати приблизно так.

## Крок 5 — Збереження контактів

Застосунок повинен зберігати масив контактів між оновленням сторінки у **localStorage**.  
Використовуй **ефекти** для синхронізації стану з локальним сховищем.

- Під час додавання та видалення контакту контакти зберігаються у **localStorage**.
- Під час завантаження застосунку контакти, якщо такі є, зчитуються з **localStorage** і записуються у стан.
